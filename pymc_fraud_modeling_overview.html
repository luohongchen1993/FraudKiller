Example model: fraud probability per transfer

P(fraud edgeᵢ = 1) = sigmoid(α + Xᵢβ + u_cluster + u_account)

Xᵢ = [amount band, 2-4AM, shared device, new account, circularity, centrality]

PyMC lets us encode fraud-ops priors, then update them with observed data. The output is not just a score; it is a distribution.

Uncertainty changes the action

Instead of a single brittle score:

Freeze if expected fraud loss exceeds customer-friction cost.

Review when uncertainty is high but evidence is strong.

Watch when posterior risk is moderate but not decisive.

Action* = argminₐ E[cost(a, fraud_state) | evidence]

What a PyMC Labs engagement would likely build

1. Feature audit

Turn graph evidence into statistical features and priors.

2. Bayesian scorer

Posterior risk for edge, account, and cluster.

3. Calibration checks

Posterior predictive checks, false-positive tradeoffs, drift tests.

4. Decision policy

Freeze/watch/clear threshold based on expected cost.

5. Agent artifact

Write model outputs into Cognee for agentic summaries.

6. Monitoring

Track base-rate shifts and pattern drift over time.
